# Migration: do-it 0.4.x → 0.5.0

## What changes

- **New skills**: `do-it-context`, `do-it-grill-log`. Installed automatically.
- **Hook output is shorter**: `router.sh` and `grill-prompt.sh` emit ≤80-token system-reminders by default; the full skill flow is delivered when you load the corresponding `do-it-*` skill explicitly.
- **Same-session de-dup**: grill emits at most once per session unless the user asks for a re-grill (`重新 grill`, `re-grill`, `再 pressure-test`, …) or escapes (`/do-it-skip`).
- **Question / discussion mode**: prompts ending in `?` / `？` / `吗` / `呢`, or matching the question hints, are auto-classified Light and never gate at closeout time.
- **Keyword tables moved**: `hooks/lib/keywords.sh` is now a thin loader; the tables themselves live in `hooks/data/*.tsv`. Project-level overrides via `.do-it/keywords.local.sh` continue to work (still appends `+=` to the same `DO_IT_*` arrays).
- **Tier classification tightened**: single-character CJK intent verbs (`做`, `改`, `加`, `写`, `修`, `审`, `搭`) no longer fire on their own; ASCII verbs use word-boundary matching (`fix` no longer matches `prefix`); Heavy tier requires ≥2 heavy signals to upgrade.
- **`DO_IT_DEBUG=1`**: hooks emit one stderr line per decision so you can tell why a tier / trigger fired. Default is silent.
- **Session state JSON**: each session writes hook invocation counters and tier history to `${CLAUDE_PLUGIN_DATA}/sessions/<id>/state.json`. `do-it doctor --session=<id>` pretty-prints it.
- **`verification-gate`** now reads only the *last* assistant message for completion language, passes through turns with no Edit/Write/MultiEdit calls, and respects question / discussion turns.

## What you have to do

Nothing — `do-it install` detects 0.4.x install state and silently migrates:

```bash
do-it install --target=claude       # silent migrate happens here
do-it install --target=codex        # same
```

Behaviorally:

1. State at `${installRoot}/.do-it-install-state*.json` is copied to `*.pre-migrate.json`.
2. Manifest-declared migration actions are applied (currently the 0.4→0.5 migration is action-less — additions only).
3. Normal install proceeds, which updates state version to `0.5.0`.

If you specifically *want* to pin to 0.4.x, use `--no-migrate` — install will refuse to silently upgrade and exit with code 2.

If anything goes wrong, restore from the `.pre-migrate.json` backup:

```bash
cp ~/.codex/.do-it-install-state.pre-migrate.json ~/.codex/.do-it-install-state.json
```

## Verifying the migration

After running install:

```bash
do-it doctor --target=<codex|claude>
# expected: ok: <N>, missing: 0, drift: 0
```

If `drift: 1` appears with reason "install state version …", run install once more — the migration sets the new version on success.

## Behavioral changes worth knowing

- **First `do-it-grill` call sets `state.grilled=1`.** Subsequent prompts in the same session don't trigger the grill template again. Override with: `重新 grill`, `re-grill`, `再 pressure-test`, `重新审视`.
- **Questions are now Light.** "你觉得 X 怎么样？" used to fire grill via intent-verb detection; now it explicitly maps to Light + `state.grilled=skip-question`.
- **Verification-gate is more permissive.** Discussion turns and questions don't get gated. If you want stricter gating in your project, write a project-level override and bump tier defaults locally.
- **New keyword overrides**: `<cwd>/.do-it/keywords.local.sh` still works exactly like before — it appends to the same `DO_IT_*` arrays.

## Reverting

If you really need to roll back to 0.4.x:

```bash
npm install -g @tdwhere/do-it@0.4.0
do-it install --target=<codex|claude> --no-migrate    # forces explicit confirmation
# or restore from pre-migrate backup as shown above
```
