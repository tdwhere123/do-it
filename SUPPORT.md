# Support

Use GitHub Issues for reproducible bugs, workflow improvement proposals, and
documentation gaps.

Before opening an issue, collect:

- do-it version or commit SHA;
- host: Codex, Claude Code, or both;
- install source: npm, GitHub, local checkout, or tarball;
- relevant command output from `do-it doctor`, `npm test`, installer runs, or
  hook logs;
- a short explanation of the real workflow problem.

For security issues, use [SECURITY.md](./SECURITY.md) instead of a public
issue.

For general discussion, use GitHub Discussions:

https://github.com/tdwhere123/do-it/discussions
